Visual Programming, How to setup up yourself for the future of BIM

by Godfrey Muhinda
November 22, 2022
What is Visual Programming?

Visual Programming is a type of programming language that enables humans describe processes(usually computer processes) using illustrations, that resemble flowcharts. The illustrations describe a logical flow of executions of a program from start to finish. Unlike a text-based programming language, whose primary purpose is to get the programmer to think like a computer, a visual programming language gives the programmer the ability to write a program in a manner that will make sense to a human being. The logical processes tend to simplify the programming language, making it easier to follow through.

Visual Programming and BIM Software

BIM Software has evolved throughout the years and with every emerging technology such as Visual Programming. The main purpose behind visual programming in BIM was to provide a platform for BIM users of all shapes and form (i.e.beginners, experienced), where they could be able to explore ideas that would improve BIM software functionality. It was meant to allow a user to create a program that does a particular job, without ever learning any text-based form of programming. It removes the need to learn text-based programming for BIM users who might not have a programming background and who have no interest in moving towards that direction. Visual Programming within BIM is achieved using several toolkits, the two most common being DynamoBIM for Autodesk Revit and Grasshopper for Rhino3D


Understanding the Visual Programming process

Before we delve any further in setting yourself up with Visual programming, it would be interesting to understand how the entire programming process works. For this example we will talk about Revit and all the tools associated with it. We obviously wouldn't want to go as far back as when Revit started, don't get me wrong, but just to understand the language Revit was built with and how Revit has been extended into Visual Programming.

Revit is primarily written in the C++ programming language, owing to its benefits such as faster execution time and easy integration with the computer's hardware resources such as the Graphical Processing Unit. C++, also used in video-game programming, gives Revit its 3D Modelling and Rendering capabilities. Without it, it would be difficult to execute 3D modeling tasks as they require a lot of hardware utilization. There are some disadvantages to using C++, which may not be discussed in this forum, but you can get more information here - Disadvantages of using C++.

Revit API, SDK & Dynamo

Now that we have some understanding of the language behind Revit, it makes sense to ask, so how does Visual Programming really work with Revit? Well to answer this question, we will go back to the whole C++ language and how one creates software for an already existing software. Most software, both online in the form of web applications, and offline in the form of desktop applications etc, usually have what we call a Standard Development Kit or SDK. This SDK has a set of tools or in programming terminology, "libraries" that can be borrowed from time to time to execute a specific task. This libraries act as a form of connection or "interface" to the original language with which the software was built. In our case, we have C++ as the base language, and for the libraries, we have a set of other languages such as C#, Python, VB.NET, and Ruby. The availability of a Standard Development Kit give birth to what we call an Application Programming Interface or API in short.


It definitely feels like we have gone too deep in this, but trust me, understanding how this whole setup works is crucial for your success with Dynamo. Let me continue and try my best to shed light on this. Don't worry, alittle programming knowledge won't hurt!

So there we have it. We now know the base language - C++, and we also now know the languages used to connect to the base language, C#, Python, VB.NET and Ruby. We are now also aware of the libraries or Standard Development Kit, developed for this connecting languages. So where is Dynamo in all this? Fear not, we are getting there. I just need to get you to see how all this information connects in the long run.

Visual Programming - A specialized Interface tool

Most newbies and experienced BIM Revit users would shy away from the idea of programming, owing to its complexity and that learning programming would be too much of a hassle. The Standard Development Kit was developed for programmers to create additional functionality and increase Revit's capabilities. This meant that those who would want to do the same but did not have the required programming skills, were easily locked out from doing so. In comes Visual programming, and that's where things got interesting. Visual programming gives the non-programmer the ability to create software just like a programmer would. To achieve this, the user would install a tool such as Dynamo onto Revit, allowing them to be able to create Visual programming scripts or short codes. This tool, Dynamo, has the capabilities of interacting with the Revit API and SDK in a manner similar to what a programmer would do. Its simply a bridge between the user and the SDK libraries and tools.

Visual Programming - The Setup

We now have some understanding of the background behind Visual Programming and Dynamo, and how it interacts with Revit. The next step is to get ourselves up and running. The steps are as follow:

  • Download – To download the official released version of Dynamo, visit the Dynamo Website. Start the download immediately by clicking from the homepage or browse to the dedicated download page. Downloads will apply for those with earlier Revit versions, from year 2019. Dynamo comes installed with the latest Revit versions, 2020 and onwards.
  • Installation – Browse to the directory of the downloaded installer and run the executable file. During the installation process, the setup allows you to customize the components that will be installed, as shown below

  • Launching – To launch Dynamo, browse to \Program Files\Dynamo\Dynamo Revit\x.y, then select DynamoSandbox.exe. This will open the stand-alone version and present Dynamo's Start Page. On this page, we see the standard menus and toolbar as well as a collection of shortcuts that allow us to access file functionality or access additional resources.

  • Opening Sample File – Open the first sample file to open your first workspace and confirm Dynamo is working correctly. Click Samples > Basics > Basics_Basic01.dyn. If this file successfully load, you should be able to execute your first Visual Program with Dynamo. For further details, please visit the site, DynamoBIM.org

Dynamo Packages - You need them!

Dynamo is an open source visual programming platform, allowing both programmers and non-programmers to create their own libraries and tools that can be shared with others. There are currently a number of useful, ready-to-go tools (visual scripts), that have already been created by other people that you can use repeatedly in your projects. Most of these tools are accessible via what we call Dynamo Packages. The biggest advantage of using these tools is the time-saving aspect, as most of these Dynamo Packages are likely to have the tools that you are looking for.

So which are the most commonly used packages out there and how do I get access to them? Here are some of the most commonly used Dynamo Packages.

  • ArchiLab - archi-lab is a collection of over 50+ custom packages that vastly extend Dynamo's ability to interact with Revit.Nodes contained in archi-lab package vary from basic list operations to advanced Analysis Visualization Framework nodes for Revit.

  • Bimorph Nodes - BimorphNodes is a versatile collection of powerful utility nodes. The package highlights include ultra-efficient clash detection and geometry intersection nodes, ImportInstance (CAD) curve conversion nodes, and linked element collectors that resolve limitations in the Revit API.

  • Bumble bee - Bumblebee is an Excel and Dynamo interoperability plugin that vastly improves Dynamo’s ability to read and write Excel files.

  • Clockwork - Clockwork is a collection of custom nodes for the Dynamo visual programming environment. It contains many Revit-related nodes, but also lots of nodes for various other purposes such as list management, mathematical operations, string operations, unit conversions, geometric operations (mainly bounding boxes, meshes, planes, points, surfaces, UVs and vectors) and paneling.

  • DataShapes - DataShapes is a package that aims to extend the user functionality of Dynamo scripts. This has a heavy focus on adding greater functionality to Dynamo player .

  • DynamoSAP - DynamoSAP is a parametric interface for SAP2000, built on top of Dynamo. The project enables designers and engineers to generatively author and analyze structural systems in SAP, using Dynamo to drive the SAP model.

  • Dynamo Unfold - This library extends Dynamo/Revit functionality by enabling users to unfold surface and poly-surface geometry. The library allows users to first translate surfaces into planar tessellated topology, then unfold them using Protogeometry tools in Dynamo.

  • Dynastrator - Import vector art from Illustrator or the web using .svg. This allows you to import manually created drawings into Dynamo for parametric operations.

  • Energy Analysis for Dynamo - Energy Analysis for Dynamo allows the user to configure the energy model from Autodesk Revit, submit to Green Building Studio for DOE2 energy analysis, and dig into the results returned from the analysis.

  • Firefly - Firefly is a collection of nodes which enable dynamo to talk to input/output devices, like the Arduino micro controller. Because the data flow happens “live”, Firefly opens up many opportunities for interactive prototyping between the digital and physical worlds through web cams, mobile phones, game controllers, sensors and more.

  • Lunchbox - LunchBox is a collection of reusable geometry and data management nodes. The tool includes nodes for surface paneling, geometry, Revit data collection, and more!

  • Mantis Shrimp - Mantis Shrimp is an interoperability project that allows you to easily import Grasshopper and/or Rhino geometry into Dynamo.

  • Mesh Toolkit - The Dynamo Mesh Toolkit provides many useful tools for working with mesh geometry. The functionality of this package includes the ability to import meshes from external file formats, generate meshes from pre-existing Dynamo geometry objects, and manually build meshes through vertices and connectivity information.

  • Monocle - Monocle is a View Extension for Dynamo 2.0.x. Monocle contains a set of useful tools for package identification, graph cleanup and more! Monocle aims to add functionality to the Dynamo UI in a seemless way that leaves you thinking, "is this built into dynamo?".

  • Optimo - Optimo provides dynamo users with the capability to optimize self-defined design problems by using various evolutionary algorithms. Users can define the problem objective or set of objectives as well as specific fitness functions.

  • Rhynamo - The Rhynamo node library provides users with the ability to read and write Rhino 3DM files from within Dynamo. Rhynamo translates Rhino geometry into usable Dynamo geometry by using McNeel’s OpenNURBS library allowing for new workflows that can exchange geometry and data fluidly between Rhino and Revit.

  • Rhythym - Rhythm is a set of useful nodes to help your Revit project maintain a good rhythm with Dynamo. basically it does some pretty okay stuff. Rhythm is open source and primarily built in C#, and adds Revit nodes, core nodes and a view extension to your Dynamo.

  • Spring Nodes - Spring nodes main focus is to improve Dynamo's interaction with Revit. The wider goal is to explore any and all means that can help accelerate BIM focused work-flows. Many of the nodes use either IronPython or DesignScript and can be a good starting point for learning the specific syntax and finer points of both.

Ready for Lift-off!!

You are now ready to go, and all the tools you need to move are within your reach. The next step is to learn how to use the software and to take advantage of its full capabilities. Be sure to check out more videos on what Dynamo can do for you, at the Autodesk University.
